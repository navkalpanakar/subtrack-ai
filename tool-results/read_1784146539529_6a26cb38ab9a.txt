    69→
    70→Work Log:
    71→- Diagnosed via user's screen recording (extracted 7 frames with ffmpeg, analyzed with VLM): sequence was login → tap demo → spinner → back to login, repeating. Never reached dashboard.
    72→- Dev log confirmed: POST /api/auth/callback/credentials 200 (login ok, cookie SET) → GET /api/auth/session returns null (cookie NOT sent back) → loop. Repeated 4x then redirected to signin.
    73→- Root cause: preview panel renders the app in a CROSS-ORIGIN iframe. NextAuth's session cookie is SameSite=Lax by default, which browsers block in cross-site iframes (third-party cookie blocking). Cookie gets set but immediately dropped, so every subsequent request is unauthenticated.
    74→- Fix: implemented a TOKEN-BASED session that bypasses cookies entirely:
    75→  * src/lib/token-store.ts — server-side in-memory Map (token → userId), 7-day TTL, survives hot reloads.
    76→  * POST /api/auth/demo-login — creates/gets user + seeds demo subs, issues token, returns {token, user}.
    77→  * GET /api/auth/me — validates token from x-subpilot-token header, returns user.
    78→  * POST /api/auth/logout — revokes token.
    79→  * src/lib/session.ts getUserId(req) — checks header token first, falls back to NextAuth session (Google OAuth still works in production).
    80→  * All 10 API routes updated to pass `req` to getUserId().
    81→  * src/hooks/use-auth.tsx — AuthProvider with global fetch wrapper that injects x-subpilot-token header on EVERY request from localStorage. signInDemo/signOut manage the token. useAuth() hook.
    82→  * RootGate, LoginScreen, AppShell rewritten to use useAuth() instead of useSession().
    83→  * Kept NextAuth for Google OAuth (production path with real credentials); only the demo flow uses tokens.
